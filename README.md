## Overview

NSI Premiere Site is a study-focused website created for the NSI Premiere curriculum. It gathers course explanations, practice exercises, and hands-on TP resources into a single student-friendly interface, so learners can review material and apply it without switching between tools.

The layout is meant for students who want quick access to class content and a practical path through the activities. It works well for classroom sessions as well as self-directed revision, with the information arranged around the needs of the NSI Premiere program.

## Installation

Clone the repository or download the source files, then open the site in a web browser.

```bash
git clone https://github.com/chrisx61/nsi-premiere-study-site.git
cd nsi-premiere-site
```

If you are serving it locally, open the main HTML entry point from the project folder or use a local static server.

## How to use it

Open the course pages, choose the topic you want to work on, and continue into the matching exercises and TP sections.

Typical workflow:

1. Open the homepage in your browser.
2. Choose the chapter or topic you want to study.
3. Read the course material.
4. Work through the exercises or practical labs.
5. Return to the site whenever you need revision material.

For local testing, you can serve the folder with any static web server and refresh the page after changes.

## FAQ

**How do I make changes to the site?**  
Edit the HTML files in the repository, then redeploy the updated version or reopen the changed files.

**Where can I find the exercises and TP content?**  
They are placed in the site pages alongside the course sections so they are easy to reach.

**Is offline access possible?**  
Yes. If the files are stored locally, you can open them in a browser or run them through a local web server.

**What should I check if a page looks broken?**  
Verify file paths, inspect the browser console, and confirm that all linked assets are referenced correctly.

**What is the best way to update the content?**  
Work directly in the HTML and keep the existing site structure intact so navigation stays simple.
